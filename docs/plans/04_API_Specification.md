# OmniTrade: API Specification

This document details the REST API surface provided by the Go (`go-chi`) backend. This API bridges the React 19 / Vite frontend with the Intelligence and Data Planes.

## 1. Authentication & Base URL
- **Base URL:** `/api/v1`
- **Authentication:** All routes (except `/auth/login`) require a valid JWT passed in the `Authorization: Bearer <token>` header.

## 2. API Endpoints

### 2.1 Authentication & User Space
#### `POST /auth/login`
- **Description:** Authenticates a human operator.
- **Request Body:** `{ "email": "user@example.com", "password_hash": "..." }`
- **Response:** `{ "token": "jwt_string", "user_id": "uuid", "role": "trader|admin" }`

#### `GET /user/profile`
- **Description:** Returns the current user's profile and permissions.

### 2.2 Market & Portfolio Data (Data Plane)
#### `GET /market/portfolio`
- **Description:** Returns the current aggregated portfolio value and a list of held assets.
- **Response:** 
  ```json
  {
    "total_value_usd": 150500.25,
    "assets": [
      { "symbol": "AAPL", "shares": 150, "current_price": 225.40, "unrealized_pnl": 1250.00 }
    ]
  }
  ```

#### `GET /market/ticker/{symbol}`
- **Description:** Returns the latest price, 24h volume, and MACD/RSI indicators for a specific asset.

### 2.3 Trade Proposals & HITL (Action Plane)
#### `GET /proposals/pending`
- **Description:** Fetches the queue of trade proposals generated by the Genkit Portfolio Manager that require human review.
- **Response:**
  ```json
  [
    {
      "id": "uuid-1234",
      "symbol": "TSLA",
      "action": "SELL",
      "proposed_by_model": "deepseek-coder-v3",
      "confidence_score": 0.88,
      "chain_of_thought": "Risk manager flagged macro volatility. Quant agent noted RSI divergence...",
      "created_at": "2026-03-04T10:00:00Z"
    }
  ]
  ```

#### `GET /proposals/{id}/references`
- **Description:** Fetches the underlying MinIO document chunks (e.g., 10-K paragraphs, news articles) that the RAG pipeline used to justify the trade.

#### `POST /proposals/{id}/approve`
- **Description:** Executes a pending trade. This requires strong authentication.
- **Request Body:** `{ "signature": "cryptographic_proof", "notes": "Looks good." }`
- **Response:** `{ "status": "executed", "execution_price": 245.50, "audit_id": "uuid" }`

#### `POST /proposals/{id}/reject`
- **Description:** Discards a pending trade and logs the rejection reason.
- **Request Body:** `{ "reason": "Too much exposure to EV sector right now." }`

### 2.4 Intelligence & Agent Config
#### `GET /agents/config`
- **Description:** Returns the currently active LLM models assigned to each agent role.
- **Response:**
  ```json
  {
    "fundamental_summarizer": { "provider": "ollama", "model": "nomic-embed-text-v1.5" },
    "portfolio_manager": { "provider": "anthropic", "model": "claude-4-6-opus" }
  }
  ```

#### `PUT /agents/config`
- **Description:** Allows an Admin to hot-swap the LLMs backing specific agents (e.g., changing the Data Analyst from Mistral to Llama 3).
- **Request Body:** (Same schema as the `GET` response).
